Export for Zoho Books

Bank-import CSV uses Double-Column layout (separate Withdrawals / Deposits) with date as dd/MM/yyyy. In Zoho choose this exact date format on the Map-Fields screen.

How to actually get this into Zoho (no passwords)

  1. In Zoho Books: Banking → Add Bank Account → add this Kotak savings A/c manually.
  2. Open the account → Import Statement (gear / top-right).
  3. Upload the Zoho Bank-Import CSV, pick Double Column, map fields, set date dd/MM/yyyy, Import.
  4. Create the ledgers from the Chart-of-Accounts CSV under Accountant → Chart of Accounts.
  5. In Banking, categorise each imported line to its ledger (your Coded CSV is the cheat-sheet).

For true API automation later, generate a Self-Client OAuth token in Zoho's API console — never paste your Zoho login anywhere. Ask me and I'll write you a small server-side script that uses that token.

Read before you file

  • Transfers ≠ expenses. Every Sweep / FD / RD line is money between your own accounts (ActivMoney 3,82,000 → 4,35,000). Booked as Transfer / Contra — keep them out of P&L.
  • Row 9 & 10 cancel out — a ₹21,658 UPI that was reversed the same day (REV-UPI). Net zero; both marked Contra.
  • Investments: Zerodha / Stable Broking money-out is an asset purchase, money-in is a redemption — not income/expense.
  • Loan/CC repayments (Fibe, LazyPay, OneCard, Credit Card Bill) reduce a liability — only the interest portion is an expense.
  • ⚠ amber dots = I'm guessing. Recurring small payees (Sunil Kumar, Dharmendra, Deepak Bhatia, M S Green Chill) and the ₹172,062 RT Carpets credit need your call. RT Carpets may have deducted TDS — track it as TDS receivable.
  • GST input credit applies only to business purchases backed by a tax invoice with a GSTIN — UPI to individuals carries none. Confirm all GST/TDS with your CA.
