How it works
  • Matching. The first 20 characters of the Description are the card network's merchant field. The tool keys on that (digits/punctuation ignored), so appended notes or changing invoice numbers don't break recognition.
  • Auto: merchant seen before with exactly one historical mapping → filled silently. Choose: merchant has more than one historical mapping (e.g. Amazon as snacks vs IT) → never guessed, you pick from its history. Guess: never-seen merchant → vendor/keyword heuristics propose values, flagged amber for your review.
  • Learning. When you download, every confirmed row (auto, kept, or edited/verified by you) is added to memory in this browser (localStorage). Unverified guesses are not learned. Use Export/Import memory to move it between machines or keep a backup.
  • Safety. The output file is your original workbook with only the Item/Vendor/Department cells rewritten inside the sheet XML — formulas (vendor & item internal-ID lookups, amounts), all other sheets, styles and dropdown validations are byte-for-byte preserved. A full recalculation is requested on open so the lookup columns refresh.
  • Dropdowns. Editing a cell offers the same lists as the workbook's own validations: Item Master column B, Vendor master column I, Departments column A.
